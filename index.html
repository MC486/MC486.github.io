Project Overview

This project represents an ambitious exploration into the intersection of game development and artificial intelligence. At its core, it's an AI-powered word game that challenges players with an opponent that learns and adapts to their strategies over time. The game combines traditional word game mechanics with sophisticated machine learning techniques to create a dynamic and engaging experience.

What sets this project apart is its hybrid AI approach, which combines multiple machine learning models to create a more sophisticated and adaptable opponent. The system employs Monte Carlo Tree Search for strategic decision-making, Naive Bayes for pattern recognition, Markov Chains for word generation, and Q-Learning for reinforcement-based adaptation. This multi-model approach allows the AI to balance exploration of new strategies with exploitation of proven tactics, creating a challenging and engaging opponent that improves over time.

The technical implementation features a robust, event-driven architecture that cleanly separates concerns between game logic, AI decision-making, and data persistence. The system uses efficient data structures for managing game state and word validation, while maintaining a flexible architecture that supports both human players and AI components. The database layer employs a repository pattern to abstract data operations, ensuring reliable persistence of game state, player statistics, and AI learning metrics.

Q-Learning (q_learning_model.py)

This QLearningAgent class implements a sophisticated reinforcement learning system that enables the AI to learn optimal word selection strategies through gameplay experience. It maintains a Q-table mapping game states (available letters and turn number) to word choices, employing an epsilon-greedy policy to balance exploration and exploitation. The implementation includes dynamic learning rate adjustment and reward shaping based on word scores, while integrating with the game's event system to track state changes and update Q-values. This component is crucial for creating an AI opponent that improves its strategy over time, providing increasingly challenging gameplay.

Markov Chains (markov_chain.py)

The MarkovChain class implements a sophisticated probabilistic language model that enables the AI to generate contextually appropriate words based on learned patterns. It builds and maintains a transition matrix mapping letter sequences to their likely next characters, with support for configurable order parameters to control context depth. The implementation includes efficient word generation algorithms that respect available letter constraints while maintaining both start probabilities and transition probabilities. This component is essential for creating natural-sounding word choices and adapting to different game scenarios.

Monte Carlo Tree Search (mcts.py)

This MCTS class implements a sophisticated tree search algorithm that enables the AI to make strategic word selections by exploring possible game states. It employs the UCB1 formula to balance exploration and exploitation, with nodes representing partial words and edges representing available letters. The implementation includes efficient tree traversal, node expansion, and reward propagation mechanisms, while maintaining statistics about node visits and win values. This component is crucial for creating an AI that can think ahead and make strategic decisions based on potential outcomes.
